**Версия**: 1.0 | **Дата**: 21 апреля 2026 | **Статус**: Draft

## 1. ОБЗОР ПРОЕКТА

### 1.1 Цель сайта

Создать одностраничный лендинг PR-агентства AWSM, который:
- Генерирует лиды через форму бесплатного аудита
- Демонстрирует экспертизу через кейсы и цифры результатов
- Объясняет ценность гео-продвижения понятным языком
- Отражает статус агентства: современно, технологично, результативно

### 1.2 KPI и метрики успеха

| Метрика | Целевое значение |
|---------|-----------------|
| Конверсия в заявку на аудит | ≥ 3% |
| Количество заявок/месяц | ≥ 50 |
| Среднее время на сайте | ≥ 2,5 мин |
| Показатель отказов | ≤ 45% |
| Scroll depth до CTA-формы | ≥ 60% посетителей |
| Core Web Vitals LCP | ≤ 2,5 сек |
| Core Web Vitals CLS | ≤ 0,1 |

### 1.3 Допущения и ограничения

- Язык: только русский, v1.0
- Персоны основателей не выводятся (добавятся в v2.0)
- Видео-тизер — заглушка с плашкой (заменяется после производства)
- Домен: awsm.agency (placeholder до подтверждения)
- Форма заявок: Telegram webhook (плашка до настройки)
- Юридические страницы: плашки, заполняются позже

## 2. ЦЕЛЕВАЯ АУДИТОРИЯ

### 2.1 Портреты пользователей

**Персона А — Владелец салона красоты**
- Возраст: 28–45 лет, чаще женщина
- Боль: есть запись на Яндексе, но клиенты не приходят с карт
- Мотивация: стабильный поток новых клиентов
- Страх: потратить деньги и не увидеть результата
- Ключевой триггер: кейс похожего бизнеса с конкретными цифрами

**Персона Б — Владелец автосервиса**
- Возраст: 35–55 лет, мужчина
- Боль: конкуренты с картой в ТОПе — он не понимает почему
- Мотивация: заполненность бокса на 2+ недели вперёд
- Страх: сложно, долго, надо разбираться самому
- Ключевой триггер: простая схема работы + цена понятна сразу

**Персона В — Владелец кафе/ресторана**
- Возраст: 30–50 лет
- Боль: пустые залы в будни, зависимость от агрегаторов
- Мотивация: снизить комиссию агрегаторов, получать прямых клиентов
- Страх: платит много, а результата нет
- Ключевой триггер: кейс ресторана с ростом трафика в будни

**Персона Г — Владелец стоматологии / клиники**
- Возраст: 35–55 лет
- Боль: низкий рейтинг и мало отзывов отпугивают пациентов
- Мотивация: доверие пациентов, полная запись
- Страх: репутационные риски
- Ключевой триггер: работа с отзывами как часть пакета

### 2.2 User Journey

```
Реклама / Поиск / Рекомендация
        ↓
Главная страница — Hero (3 сек на первое впечатление)
        ↓
Видео-тизер или блок Проблема/Решение
        ↓
"Это про мой бизнес" — блок Кому подходит
        ↓
Доверие — Кейсы с цифрами
        ↓
Понимание цены — Тарифы
        ↓
Действие — Форма аудита (CTA)
        ↓
Подтверждение — Thank You (redirect или modal)
```

### 2.3 Jobs To Be Done

- «Хочу понять: мне это вообще нужно?» → блок Проблема/Решение
- «Хочу убедиться что это работает» → Кейсы с реальными цифрами
- «Хочу понять сколько стоит» → Тарифная страница
- «Хочу начать без риска» → Бесплатный аудит как точка входа

## 4. ДИЗАЙН-СИСТЕМА

### 4.1 Визуальная концепция

**Название концепции**: «Digital Cartography»

Метафора: город ночью с высоты птичьего полёта. Большинство бизнесов — тёмные точки, невидимые алгоритму. Клиент AWSM — яркий маяк, который притягивает трафик. Алгоритмы геосервисов визуализированы как живые потоки данных, а не скучные таблицы.

**Атмосфера**: технологично, экспертно, живо. Не холодно-корпоративный, не кричащий стартап. Агентство, которое умеет работать с алгоритмами так же, как Goldman Sachs умеет работать с деньгами — уверенно и без лишних слов.

**Ключевые визуальные решения**:
- Светлый фон (`#FAFAFE`) как профессиональная, доверительная среда. **Тёмная тема не используется — решение закреплено 27.04.2026.**
- Электрические акценты (Electric Indigo, Teal, Coral) — энергия без тёмного фона
- 3D map pins как главный визуальный мотив (Этап 8, пока в планах)
- Данные и цифры как элементы дизайна (12 000%, №1, 3 недели)
- Минимум стоковых иллюстраций, максимум типографики и данных

### 4.4 3D-элементы и WebGL-сцены

**Сцена 1 — Hero (главная)**

Описание: Тёмное пространство с плавающими 3D-пинами карт (стилизованные location markers). Несколько пинов светятся ярко (клиенты AWSM), остальные тусклые (конкуренты). Частицы соединяют пины, имитируя потоки трафика. Сцена реагирует на движение мыши (parallax).

Элементы:
- 15–20 floating location pins разного масштаба
- Линии-потоки между яркими пинами
- Ambient particle system (звёздная пыль)
- Мягкое свечение от ярких пинов (glow shader)
- Плавное вращение всей сцены по оси Y

Технология: Three.js / React Three Fiber

**Сцена 2 — Алгоритмы (блок Проблема/Решение)**

Описание: Абстрактная сфера из точек и линий, имитирующая нейронную сеть или алгоритм. При scroll — сфера «активируется»: точки разлетаются и выстраиваются в карту. Символизирует переход от «мёртвого» профиля к «живому».

Элементы:
- Wireframe sphere → explode → reorganize as map cluster
- Цвет по умолчанию: muted gray → при активации Electric Indigo + Neon Teal
- Scroll-triggered animation

Технология: Three.js + GSAP ScrollTrigger

**Сцена 3 — Метрики (блок Кейсы)**

Описание: Floating 3D цифры результатов в пространстве. +12 000%, №1, 3 недели — объёмные числа с neon glow. При hover на карточку кейса — соответствующая цифра «прилетает» в фокус.

Технология: Three.js Text Geometry / CSS 3D Transform (как fallback)

**Требования к производительности 3D**:
- Fallback для мобильных: CSS-анимация вместо WebGL
- Fallback для слабых устройств: prefers-reduced-motion → статика
- Lazy load 3D сцен (не блокировать LCP)
- Target: 60 fps на десктопе, 30 fps на мобайле

### 4.5 Система анимаций

**Уровень 1 — Базовые переходы** (всегда)
- Fade-in при появлении в viewport: opacity 0→1, translateY 30px→0, duration 0.6s
- Stagger для списков и карточек: задержка 0.1s между элементами
- Hover на кнопках: scale 1.02, glow усиливается
- Hover на карточках: translateY -4px, border color меняется на Primary Accent

**Уровень 2 — Scroll-triggered** (GSAP ScrollTrigger)
- Счётчики цифр: animate от 0 до конечного значения при входе в viewport
- Horizontal scroll секций: кейсы и тарифы прокручиваются горизонтально на десктопе
- Pinned sections: секция "Как мы работаем" — sticky, шаги появляются по мере скролла
- Text reveal: крупные заголовки split на chars, letter-by-letter появление

**Уровень 3 — Cursor-driven** (только десктоп)
- Custom cursor: круглый dot (8px) + ring (32px), трейлинг с задержкой
- Magnetic buttons: CTA кнопки притягивают курсор при приближении (magnetic effect, ±15px)
- Cursor states: default / hover-link / hover-button / drag / hidden (на видео)
- Color change cursor при наведении на разные секции

**Уровень 4 — Page transitions** (Framer Motion)
- Переход между страницами: curtain wipe (#080C14 overlay, 0.4s ease)
- Элементы нового URL появляются поверх curtain
- Back navigation: reverse animation

**Плавный скролл**:
- Lenis smooth scroll, factor 0.08–0.12
- Отключается для мобильных (нативный scroll)

### 4.6 Иконки и иллюстрации

- Иконки: Phosphor Icons (line style) — один набор, нет микса стилей
- Кастомные иконки для категорий бизнеса: авто, красота, здоровье, еда, услуги, дети (inline SVG, Electric Indigo)
- Иллюстраций нет — заменяем данными, цифрами и 3D
- Паттерн-текстура фона: subtle dot grid (точки 1px, opacity 0.04) на тёмных секциях

## 6. ИНТЕРАКТИВНЫЕ ЭЛЕМЕНТЫ

### 6.1 Custom Cursor (Desktop only)

- Состояния: default (dot + ring) / link hover (увеличенное кольцо) / button hover (filled coral) / drag / video (hidden)
- Трейлинг: ring следует за dot с задержкой 0.08s (lerp)
- Цвет: Electric Indigo в нейтральном состоянии, Coral при наведении на CTA

### 6.2 Magnetic Buttons

- Все CTA кнопки реагируют на курсор в радиусе 60px
- Кнопка «плывёт» к курсору на ±15px max offset
- При уходе курсора — spring-back анимация
- На мобайле: отключено

### 6.3 Before/After Слайдер (страница Кейсы)

- Вертикальный разделитель с drag-handle
- Левая часть: «До» (серый/мутный)
- Правая часть: «После» (яркий, Electric Indigo tint)
- Drag или hover-следование
- Метки «ДО» и «ПОСЛЕ» fade при движении

### 6.4 Animated Counters

- Активируются при входе элемента в viewport (Intersection Observer)
- Анимация: от 0 до конечного значения, ease-out, 1.5–2s
- Для «+12 000%»: анимируется число, знак + и % статичны
- Шрифт: JetBrains Mono, цвет Neon Teal

### 6.5 Scroll-triggered Text Reveal

- Крупные заголовки: split на символы (SplitType library)
- Chars появляются снизу вверх, stagger 0.02s на символ
- Trigger: при пересечении 20% viewport

### 6.6 Pricing Cards Hover

- При hover: карточка поднимается (-8px translateY)
- Граница меняет цвет на Electric Indigo с glow
- Внутри карточки: кнопка анимируется (slide in)
- Featured карточка: постоянная пульсирующая рамка (keyframe animation)

### 6.7 Navigation Hover

- Ссылки в nav: при hover — подчёркивание растёт слева, Electric Indigo
- Активная страница: постоянное подчёркивание
- Logo: при hover — subtle glow вокруг

### 6.8 Form Interactions

- Поля: border меняется на Electric Indigo при focus
- Label float-up при вводе (floating label pattern)
- Кнопка Submit: при hover — gradient shift, при клике — loading spinner
- После отправки: success state с анимацией checkmark

### 6.9 Page Loader

- При первом визите: короткий loader (0.8s max)
- Логотип AWSM появляется по буквам: A → W → S → M
- Затем curtain поднимается, открывая Hero

| Хостинг | Vercel | Идеально для Next.js |

| Метрика | Цель | Как достичь |
|---------|------|-------------|

| FID / INP | ≤ 200ms | Code splitting, no heavy scripts on main thread |
| CLS | ≤ 0.1 | Явные размеры всех изображений |
| TTI | ≤ 3.5s | Bundle splitting, defer non-critical JS |
| Lighthouse Score | ≥ 90 | Performance budget |

- Canonical URLs
- Sitemap.xml
- Robots.txt
- Alt-атрибуты для всех изображений
- H1 уникальный на каждой странице
- Semantic HTML (header, main, section, article, footer)
